Pipeline workflow

Each rank is a hard contract: features cannot violate the point-in-time invariant, conformal layers cannot ship without a coverage report, the release gate cannot fire without a verifiable reproducibility envelope. Two parallel branches at the regime layer (change-point and regime posterior) feed one head layer.

Composite forecast distribution

F̂t,h(y) = Ch,R ⎡ ∑m wm,t,h(CPt, γt, Rt, Lossm, CalErrm) · Fm,t,h(y) ⎤

CPt

online change-point probability (NIW BOCPD, BOCPD-MUSE, or GP-BOCPD; covariate-conditioned hazard optional)

γt

latent regime posterior (HMM Baum-Welch or MS-VAR Hamilton-Kim)

Rt

WFST-decoded regime state

wm,t,h

online BMA weights — exponentially-discounted log-score; floor applied after normalisation (v1.2)

Ch,R[·]

regime-conditional conformal layer; backend ∈ {split, block, nexcp, conditional, localized, e_conformal}. Optionally extended with Bonferroni multi-horizon for joint coverage.

Lossm, CalErrm

rolling validation loss and calibration error per model — feed the BMA weight update

Anatomy of a single forecast

How a forecast at t = 2024-12-01, horizon = 12m is produced. Every step is fail-closed: if the audit fires, the run is rejected before the forecast is emitted.

Step Module / function Math · invariant
1. Materialise PIT panel asof.materialize_feature_asof_values observation_date ≤ t and vintage_date ≤ t for every row
2. Audit lineage asof.audit_feature_asof_lineage (fail-closed) violations = 0 → continue · else SystemExit(2)
3. Build features features.build_features + robust_stats + dfm.DFMDomainModel.transform v1.2: cached training (μ, σ) → factor scale stable across panels
4. Score change-point bocpd.MultivariateNIWBOCPD or bocpd_muse.BOCPDMuse or frontier.gp_cpd.GPBOCPD CP_t = P(run-length = 0 | x_{1:t})
5. Score regime posterior hmm.HMMRegimePosterior + msvar.MarkovSwitchingVAR + wfst.RegimeWFST γ_t via Baum-Welch / Hamilton-Kim filter; R_t via Viterbi
6. Run heads models.QuantileReturnModel + hazard_model.DiscreteTimeHazardModel + frontier.distributional.* + frontier.neural_seq.PatchTSTHead non-crossing repair on quantiles; path-aware horizon survival on hazard
7. Calibrate + conformalise calibration.PlattCalibrator → conformal.MondrianBinaryConformal(backend=…) marginal coverage 1−α conditional on regime bucket (block / NexCP / Gibbs-Cherian-Candès / Lin / Vovk-Wang)
8. Mix bma.OnlineBMA.mix F̂_{t,h}(y) = ∑_m w_{m,t,h} · F_{m,t,h}(y); floor applied post-norm
9. Validate against baselines walk_forward.PurgedWalkForward + forecast_compare.{DM, GW, MCS, CRPS-DM} DM-significant beat at 5% AND MCS membership (T_R or T_SQ) OR e-value ≥ 1/α
10. Governance gate release_gates.evaluate_release_gate(promotion_method=…, min_coverage=…) fail-closed on confidence / drift / invalidation / coverage / promotion
11. Persist run model_runs.create_model_run with full ReproEnvelope git SHA · lockfile sha256 · feature/output/vintage payload sha256 · RNG seeds
12. Verify mre verify-run --run-id <id> exit 0 with approved: true ⇔ envelope round-trips bit-for-bit

v1.2 frontier package

Optional dependencies (statsmodels, ngboost, torch) live behind the [frontier] extra. Every module degrades gracefully when its soft dep is missing — the engine never breaks; it just falls back to the closest pure-numpy primitive.

Promotion gates · two complementary mechanisms

Hansen MCS (fixed-window)

release_gate(require_mcs_membership=True)

Stationary block-bootstrap with recentered null. v1.2 adds the TSQ statistic so high-power detection of marginally-tied alternatives is reachable.

T_R T_SQ block bootstrap

Hansen-Lunde-Nason 2011

Sequential e-value safe-test

release_gate(promotion_method="e_values")

Anytime-valid: stop sampling whenever the cumulative e-value crosses 1/α. The candidate is promoted on cumulative evidence, not on a fixed-window p-value.
